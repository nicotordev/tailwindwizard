üõ°Ô∏è Asset Protection Strategy

The biggest challenge in frontend marketplaces is code theft. Since HTML/CSS is sent to the browser, we employ a multi-layered security approach to protect seller assets.

Layer 1

Preview Obfuscation

Live previews utilize Shadow DOM encapsulation and randomized class names, making "Inspect Element" copying extremely difficult and functionally useless without the source utility map.

Layer 2

Server-Side Delivery

The actual `jsx/tsx` source code is never sent to the browser during preview. It resides in a secure S3 bucket, released only via a signed URL upon successful Stripe webhook confirmation.

Layer 3

Digital Watermarking

Purchased components include a unique transaction hash in the comment header. If a component leaks publicly, we can trace it back to the original buyer account for enforcement.
